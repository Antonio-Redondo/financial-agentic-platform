📥 Ingestion Pipeline

A drop-folder ingester that walks DOCUMENTS_DIR, extracts text from a dozen file formats, scans for PII, then chunks → embeds → upserts into pgvector. It's built to run on every startup: a content-hash check means unchanged files are skipped and only new or edited documents do real work.

src/ingestion/pipeline.py src/ingestion/loaders.py folder scan → extract → upsert

The drop folder

Everything starts from a single directory. documents_dir() resolves it from the DOCUMENTS_DIR environment variable, defaulting to ./data/documents, and returns an absolute path. The folder is created if it doesn't exist, so a fresh checkout just works. 

def documents_dir() -> str:
    return os.path.abspath(os.getenv("DOCUMENTS_DIR", "./data/documents"))

The walker (_iter_files) recurses the tree but deliberately skips:

  • Hidden directories — any folder starting with . (e.g. .git, .venv).
  • Hidden files — any filename starting with ..
  • The folder's own README.md — matched case-insensitively so it isn't ingested as content.

Ingestion runs in two situations: on app startup, and again whenever the sidebar triggers a re-scan after a new upload. Uploads from the UI go through save_upload(), which writes the UploadedFile into the same folder and then the scanner re-loads it from disk — so the startup path and the upload path share identical code.

🛡️ Path-traversal guard on uploads

save_upload() strips path components from the uploaded name with os.path.basename (after normalising \ to /), rejects "."/"..", and verifies with os.path.commonpath that the target stays inside the docs folder — so a crafted name like ../../etc/passwd can't escape the directory.

A single file can also be ingested directly via ingest_file(vector_store, path), which returns the number of chunks written (or None for an empty / non-file path).

Per-format loaders

load_text(path) looks up the lower-cased extension in the _LOADERS table and calls the matching extractor. Each loader imports its library lazily. Anything not in the table falls back to _load_text (UTF-8, then latin-1).

def load_text(path: str) -> str:
    ext = os.path.splitext(path)[1].lstrip(".").lower()
    loader = _LOADERS.get(ext, _load_text)
    try:
        return loader(path)
    except Exception as e:   # surface the cause, never crash ingest
        print(f"⚠️  Loader failed for {path}: {e}", flush=True)
        return ""

🧯 Loaders never crash the scan

Any extractor exception is caught inside load_text, logged with a ⚠️ line, and turned into an empty string — so a single corrupt file becomes a skipped_empty rather than aborting the whole folder. SUPPORTED_EXTENSIONS documents the known set, but unknown extensions still fall back to the UTF-8 text reader.

Content-hash dedupe & re-chunk on change

Dedupe is content-based, not timestamp-based. _hash_text takes the SHA-256 of the extracted text, and the scanner compares it against vector_store.known_sources() — a {path: content_hash} map fetched once per scan. 

def _hash_text(text: str) -> str:
    return hashlib.sha256(text.encode("utf-8", errors="ignore")).hexdigest()
  • Same hash for a path → skipped_unchanged, no embedding work.
  • New path or changed hash → upsert_document(...) re-chunks and re-embeds; on success the file counts as ingested and logs 📥 Ingested … → N chunk(s).
  • Re-uploads of the same name overwrite on disk, but the hash check downstream means identical content is never re-embedded.

🔑 Why hash after the PII step

The text is scanned (and possibly redacted) before hashing, so the dedupe hash always matches what actually gets embedded into pgvector. If redaction changes the text, the hash changes with it and the document is correctly re-ingested.

PII detection on ingest

Before anything is stored, each document passes through the guardrails layer: get_guardrails().scan_document(text) returns the (possibly redacted) text plus a pii_summary. When PII is found it's logged with a 🛡️ line, recorded under pii_findings in the chunk metadata, and — when GUARDRAILS_PII_REDACT_ON_INGEST=true — redacted so raw PII never lands in pgvector. 

text, pii_summary = get_guardrails().scan_document(text)
if pii_summary:
    print(f"🛡️ PII detected in {os.path.basename(path)}: {pii_summary}", flush=True)
...

Every chunk also carries source="folder_scan" and an ISO-8601 UTC ingested_at timestamp (suffixed Z) alongside filename, file_size and file_extension.

📊 Reading the scan summary

At the end of a run the pipeline prints 📊 Folder scan complete — scanned=…, ingested=…, unchanged=…, empty=…, errors=…. Per-file errors are caught, appended to IngestionResult.errors and logged with ❌ Ingestion error — one bad file never stops the rest of the folder.
