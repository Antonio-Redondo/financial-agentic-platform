🔭 Observability (LangSmith)

Every LLM and agent call can be traced to LangSmith for inspection — grouped into named runs and tagged with route, complexity and model. It's wired in once, driven entirely by environment variables, and ships off because it's the only part of this otherwise-local app that talks to the cloud.

src/observability.py LangSmith · smith.langchain.com off by default

What it gives you

When enabled, LangChain's callback system auto-captures the planner, retriever, analyst and embedding calls — no per-call code. On top of that, this project adds two conveniences so the traces are actually navigable.

  • Grouped runs. A whole query is wrapped as one named run — financial_query (and suggest_questions) — so the child LLM calls nest underneath it instead of scattering as loose top-level events.
  • Filterable metadata. Each run is tagged with route, complexity and analyst_model, so in the UI you can filter to e.g. only complex queries, or only answers written by qwen2.5:3b.
  • Latency & token visibility. Per-call timing and token usage come for free from the LangChain instrumentation — useful for spotting which step dominates a slow CPU response.

LANGSMITH_ENDPOINT=https://api.smith.langchain.com   # EU: https://eu.api.smith.langchain.com

No code changes are needed — LangChain auto-instruments the calls. Traces appear under the project name set above.

How it's wired

All the logic lives in src/observability.py, a thin layer over the langsmith SDK with three entry points:

HelperRole
@traceableRe-exported decorator that groups a function's child LLM calls under one named run. A cheap pass-through when tracing is off, so it's safe to leave applied permanently.
add_metadata(**fields)Tags the active run (route, model, …). A no-op when no run is on the stack, so it can sit in hot paths without guarding.
configure()Logs once at startup whether traces are really uploading. Makes no network calls when disabled.
tracing_enabled()Whether LANGSMITH_TRACING is truthy — drives the UI indicator.
 
# graph.py — the query is one grouped, tagged run
@traceable(run_type="chain", name="financial_query")
def stream_run(self, query, on_token, ...):
    ...
    add_metadata(route=state.get("route"),
                 complexity=state.get("complexity"),
                 analyst_model=model)        # → filterable in the LangSmith UI
# app.py — honour the env block once .env is loaded
observability.configure()                    # logs the one-line status at startup

Confirming it works

  • Startup log: 🔭 LangSmith tracing ON → project 'financial-agentic-platform'. If TRACING=true but the API key is missing, it warns instead — runs won't upload.
  • In the app: the sidebar shows a tracing indicator while it's enabled.
  • In LangSmith: a financial_query run appears per question, with planner / retriever / analyst calls nested inside and the route/complexity/model tags attached.

Privacy & cost

☁️ The one cloud dependency

Everything else in this platform runs 100% locally. LangSmith is a hosted service, so enabling it ships your queries, retrieved context and answers to smith.langchain.com. That's why it's disabled by default and stays off until you explicitly opt in — keep it off if the documents are sensitive, or point LANGSMITH_ENDPOINT at a self-hosted instance.

🪶 Zero cost when off

configure() early-returns when tracing isn't truthy, and add_metadata() is a no-op with no active run — so leaving the @traceable decorators in place adds no latency and makes no network calls in the default local-only mode.

Configuration knobs

LANGSMITH_TRACING
false — master switch; set true to upload traces
LANGSMITH_API_KEY
your lsv2_… key; required for uploads
LANGSMITH_PROJECT
financial-agentic-platform — where runs are grouped in the UI
LANGSMITH_ENDPOINT
US default · eu.api.smith.langchain.com for EU · or self-hosted URL

🤖 Headless / cron runs

Tracing follows the environment. If you run the app or the eval harness non-interactively, make sure the same LANGSMITH_* vars are present there too — otherwise traces silently won't upload.
